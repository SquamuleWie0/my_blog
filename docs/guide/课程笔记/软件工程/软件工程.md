# 软件工程

## 软件工程概述
### 1.1 软件与软件危机

**软件的发展历程**

| 阶段         | 时期         | 特点       | 问题                      |
| :--------- | :--------- | :------- | :---------------------- |
| **程序设计时代** | 1946-1950末 | 个体化工作    | 无规范、无文档                 |
| **程序系统时代** | 1960-1970初 | 作坊式小团体合作 | **爆发软件危机**（进度慢、成本高、改不动） |
| **软件工程时代** | 1970s至今    | 工程化生产    | 解决危机，强调规范与分工            |
**程序设计时代**（个体化工作）、**程序系统时代**（作坊式小团体合作）、**软件工程时代**（工程化生产）。

**一句话总结**：
因为“软件危机”爆发，所以有了“软件工程”。
### 软件危机
**软件危机**是指：计算机软件在开发和维护阶段出现遇见的严重问题

**产生原因**：
- 缺乏统一的、标准化开发过程
- 忽视软件开发前期的需求分析
- 没有完善质量保证体系

**典型表现**：
- 成本进度估计经常不准确
- 用户对软件不满意
- 软件质量不可靠
- 软件难以维护
- 软件没有适当的文档材料

**缓解**：软件危机正是通过引入**软件工程的理论**、**方法和工具**才得以缓解的。
但随着软件规模和复杂度的增长，成本超支、进度延误、缺陷率高等问题**依然存在**，只是表现形式不同。
### 1.2 **软件**

是什么软件：软件 = 程序 + 数据 + 文档
- 程序：计算任务的处理对象和处理规则的描述（就是代码逻辑，告诉计算机怎么做）
- 数据： 程序运行所需要的初始化信息（程序加工材料，用户信息、图片文字）
- 文档：开发使用维护程序所需要的图文手册
可执行部分是：**程序和数据**

**特征**：
1. **无形性**：软件是逻辑实体，看不见摸不着，只能通过运行看效果
2. 质量**无磨损性**：硬件会坏，软件不会，只会需要变更或者环境变化进行升级更新
3. 软件管理特性：没有流水线生产，本质是 “**设计开发**”，管理难度大进度质量难把控
4. 智能特性：软件依赖于人的智力和**逻辑**思维投入，自动化程度低，开发过程难以标准化

**软件的分类**：
1. 工作方式
	- 实时处理软件
	- 分时处理软件
	- 交互式软件
	- 批处理软件
2. 规模：微型、小、中、大
3. 服务对象
	- 通用软件
	- 定制软件
4. 分布式布置
5. 功能
	- 系统软件
		- 操作系统
		- 语言处理系统软件
		- 数据库管理软件
	- 支撑软件
	- 应用软件

**软件工程基本原则**：
- 用分阶段的生命周期计划资源管理
- 坚持阶段评审
- 严格产品控制
- 采用现代化程序设计技术
- 结果应能够清晰审查
- 开发组人员小而精
- 承认不断改进软件工程实践的必要性
- 建立变更控制流程

**软件工程方法学**：方法、工具、过程
方法：面向对象的分析方法和设计
工具：UML
过程：软件过程模型



## 软件生命周期
### 软件生命周期的定义
1. 定义时期
	- 问题定义
	- 可行性分析
	- **需求分析**（最重要的一个阶段）
2. 开发时期
	- 软件设计
	- 编码
	- 测试
3. 使用和维护时期
	- 维护

软件生命周期从项目计划开始，经过需求分析、设计、编码、测试，当软件产品**交付给用户并正式投入使用**的那一刻起，就进入了维护阶段，直到该软件被淘汰。
