# 2026 周报：从工程基础到 Agent 项目实践
## 第 21 周｜Agent 微服务完成接入，进入端到端联调问题集中处理阶段

2026-07-20 ~ 2026-07-22

这几天主要跟的是一个面向互动叙事 / 剧本生成场景的多智能体创作系统。我的工作覆盖 Agent 微服务、GVA 主后端和前端展示之间的联调，重点是把“内容生成 → 业务承接 → 页面预览”的链路跑通。

7 月 20 日基本在等后端服务器侧回复，同时排查具体问题，也顺了一下 gRPC 鉴权，对 gRPC 通信有了一个初步的整体认识。

7 月 21 日从早上 9 点做到晚上 11 点。Agent 微服务已经接通了，服务器侧的问题也处理了，但真正接通之后，细节问题就开始显现出来：

- LLM 返回的 JSON 还没有做拆分；
- 前端还没有按 type 做差异化渲染；
- 提示词编排一开始参考的是创作者中心字段，但游戏中心实际字段不一样。

这个问题本质上来自需求字段和业务场景没有提前对齐清楚。现在这个部分已经明确了，后面需要我去 GVA 那边确认前端实际传回来的字段，再根据规则重新组织提示词编排。规则比较复杂，不能一把梭硬写，得拆开来做。

这一周也意识到，后面还有几个智能体要做，尤其是「直接生成剧本」这一类能力，编排值得好好设计。这里涉及到世界设定、角色关系、剧情结构、分阶段生成、图片生成相关逻辑、结构化输出和前端展示方式之间的整体编排。这里更像是在做一个内容生产链路：前面要能承接用户输入和业务字段，中间要能组织文本 / 图像 / 规则之间的关系，后面还要保证生成结果能被页面稳定渲染和继续编辑。

另外我也进一步形成了一个判断：传统后端基础是 Agent 开发的重要底座，不能只局限在 Agent 层。传统后端转 Agent 很快，是因为接口、数据库、服务通信、工程结构这些能力本来就是 Agent 应用落地的底座。

**问题和风险**

- 提示词编排字段需要重新确认，不然后续容易继续返工。
- JSON 结构化输出和前端 type 渲染没有对齐，会影响页面效果和后续扩展。
- 同时一边教培一边跟项目，压力比较大，这个周是关键周。

**后续预期**

- 去 GVA / 前端侧确认游戏中心实际字段；
- 拆分提示词编排规则；
- 处理 LLM JSON 拆分和前端 type 渲染；
- 继续推进后续几个智能体；
- 在微服务里补数据库，把传统后端基本功补起来。

## 第 15 周｜准备七牛路演，并将 Agent 微服务缓存命中率提升到 28.6%

2026-06-08 ~ 2026-06-14

这一周主要准备七牛路演：路演准备、路演稿、Demo 录制稿、PPT 大纲都整理了。

技术上有一个比较明确的优化：把 Agent 微服务的缓存命中率搞上去了。核心思路是把稳定不变的内容，比如角色设定、行为规则，放到 prompt 前面当稳定前缀；把变化内容，比如场景、状态、对话，放到后面。

A/B 测试效果比较明显：缓存命中 token 从 256 涨到 512，平均命中率从 13.4% 提升到 28.6%。这个让我对 prompt caching 和 prompt 结构顺序有了更实际的理解。

同时完成了计组和计网实验，准备四级和周一面试，也集中归档了一批文档：Docker、Website、TrueTone、LangChain 等。

## 第 10 周｜重构 RepoMind，并对照 LangGraph 跑通 Agentic RAG 思路

5 月 8 日学了 7 个小时，重点看 LangGraph 官方 Agentic RAG 示例。理解了它的主流程：Agent 先判断是否需要检索，需要时通过 tool_call 调用检索工具，检索后再判断结果是否相关；相关就生成答案，不相关就改写问题后重新检索。

7 月进入真实项目节奏，开始处理 Agent 微服务、gRPC、GVA 主后端、前端展示、预览链路优化、版本收尾。这个阶段对我的倒逼很明显：不能只会 vibe coding，必须读源码、补后端、懂通信、懂数据库、懂业务链路。

- 主线：AI 应用工程化 / Agent 开发；
- 底座：传统后端、数据库、网络、gRPC、服务部署；
- 方法：真实项目驱动 + 系统补课 + 源码阅读 + 复盘沉淀；
- 目标：从“能写功能”，走向“能理解业务、设计系统、推进交付、判断方案”。

这几个月最大的变化是：我开始知道自己在做什么，也知道下一步该补什么。
